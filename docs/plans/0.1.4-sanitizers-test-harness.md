## 3. Overview & Approach

### 3.1 Strategy in one paragraph

Add a directory-scope sanitizer module toggled by the `*-debug` presets, add `doctest` to the pinned manifest, stand up a single `aero_tests` runner with one trivial test wired through six host-gated test presets, and add three CI steps (a Windows `link`-shadow guard + two `ctest` steps). Verify everything that a Mac *can* verify locally, then use the PR's CI run to prove the three things a Mac cannot: the MSVC ASan compile+link path, and that CI actually goes red on a failing check / UBSan trip / ASan trip.

### 3.2 Load-bearing sequencing & correctness constraints

These are the non-obvious "get-this-order/flag-right-or-it-silently-doesn't-work" facts. Treat each as load-bearing, never as cleanup.

1. **`include(cmake/sanitizers.cmake)` placement in the root `CMakeLists.txt`.** It must come **after `project()`** (compiler IDs — `MSVC`, `CMAKE_CXX_COMPILER_ID` — are only known post-`project()`), and **before any `add_subdirectory()`** (the module calls `add_compile_options`/`add_link_options`, which are directory-scope and only propagate to subdirectories added *after* the call). Placing it after `add_subdirectory(engine)` etc. would leave those targets — and `aero_tests` — unsanitized with no error. The only correct home is between the C++-standard block (ends line 19) and the `# Layer order` comment (line 21).
2. **`-fno-sanitize-recover=all` is load-bearing (D3).** UBSan's default is recover-and-continue: it prints a report and **exits 0**, so `ctest` passes and the lane stays green even on real UB (proven locally, spec §5e). The no-recover flag makes UBSan **abort** (exit 134) → `ctest` fails → lane red. This flag is the entire reason criterion 6's "UBSan trip reds the lane" is true.
3. **Why the manifest edit only rebuilds doctest (not a cold SDL3 rebuild).** Editing `vcpkg.json` changes `hashFiles('vcpkg.json')`, so CI's exact cache key `vcpkg-<os>-<hash>` misses. The `restore-keys: vcpkg-<os>-` prefix then restores the previous cache, so SDL3's archives come back warm and only the newly-added doctest (header-only) compiles — seconds, not minutes.
4. **Case order matters in the proof file (§7).** doctest runs `TEST_CASE`s in registration order, and a sanitizer abort kills the whole process (a process can only die once). Ordering *plain CHECK → UBSan → ASan* lets a single Debug process record the CHECK failure and then trip the first sanitizer reachable on that platform: on macOS/Linux UBSan aborts at case 2; on Windows (no UBSan) case 2 is silent and ASan aborts at case 3. Proof-2 then deletes cases 1–2 so macOS/Linux also reach the ASan case. Reversing the order would hide UBSan on mac/Linux.
5. **MSVC flag conflicts are invisible on the Mac (D2, spec §6.2/§6.3).** CMake's default `CMAKE_CXX_FLAGS_DEBUG` carries `/RTC1`, which errors when combined with `/fsanitize=address`; the module strips it with `string(REPLACE)`. Default Debug link flags carry `/INCREMENTAL`; the module appends `/INCREMENTAL:NO`, which wins because MSVC's linker honors the *last* occurrence and `add_link_options` lands after the built-in flags. Neither can be checked on macOS — the **Windows Debug CI lane building at all** is the proof (an unstripped `/RTC1` would make `cl` error loudly).
6. **`enable_testing()` must precede `add_subdirectory(tests)`.** It is emitted in the same root insertion (step S2), which sits above all `add_subdirectory` calls — satisfied automatically.

### Commit 1 — `build:` (sanitizers module, root CMake, presets, manifest)

#### S1 — Create `cmake/sanitizers.cmake` (new file; creates the `cmake/` directory)

Verbatim (spec §3.2):

```cmake
# cmake/sanitizers.cmake — ASan/UBSan wiring (ADR-001 mitigation #3; docs/04).
# Enabled by the *-debug presets via AERO_ENABLE_SANITIZERS (OFF by default).
# Included from the root CMakeLists BEFORE any add_subdirectory(), so the
# directory-scope flags below reach every project target. vcpkg dependencies
# are NOT affected (they build in their own toolchain invocation); ASan
# tolerates uninstrumented code, so linking them stays safe.

if(NOT AERO_ENABLE_SANITIZERS)
    return()
endif()

if(MSVC)
    # MSVC ships ASan only — /fsanitize=undefined does not exist (spec 0.1.4
    # §5a). ASan is incompatible with /RTC1 (present in CMake's default Debug
    # flags) and with incremental linking, so strip the former and force the
    # latter off. MSVC's linker honors the LAST /INCREMENTAL option, and
    # add_link_options() lands after the built-in Debug link flags.
    add_compile_options(/fsanitize=address)
    string(REPLACE "/RTC1" "" CMAKE_CXX_FLAGS_DEBUG "${CMAKE_CXX_FLAGS_DEBUG}")
    add_link_options(/INCREMENTAL:NO)
elseif(CMAKE_CXX_COMPILER_ID MATCHES "Clang|GNU")
    # AppleClang matches "Clang". -fno-sanitize-recover=all is load-bearing:
    # without it UBSan prints its report and exits 0, and CI stays green
    # (spec 0.1.4 §5e). Frame pointers keep ASan stacks readable.
    add_compile_options(
        -fsanitize=address,undefined
        -fno-omit-frame-pointer
        -fno-sanitize-recover=all
    )
    add_link_options(-fsanitize=address,undefined)
else()
    # Mitigations are enforced, not encouraged (docs/04): an unmapped compiler
    # must fail loudly rather than silently build unsanitized.
    message(FATAL_ERROR "AERO_ENABLE_SANITIZERS=ON but compiler '${CMAKE_CXX_COMPILER_ID}' has no sanitizer mapping (cmake/sanitizers.cmake)")
endif()
```

### Proof-1 commit

```cpp
// red_proof_test.cpp — DO NOT MERGE (0.1.4 acceptance criterion 6).
// Case order matters: a sanitizer abort kills the process, so UBSan must
// fire before ASan (doctest runs cases in registration order).
#include <doctest/doctest.h>
#include <climits>
#include <cstdlib>

TEST_CASE("red-proof 1: plain failing check") {
    CHECK(false);                       // reds every lane, both configs
}

TEST_CASE("red-proof 2: UBSan signed overflow") {
    int k = INT_MAX;
    k += 1;                             // aborts on macOS/Linux Debug; silent on MSVC (no UBSan) and in Release
    CHECK(k < INT_MAX);
}

TEST_CASE("red-proof 3: ASan heap-use-after-free") {
    int* p = static_cast<int*>(std::malloc(sizeof(int)));
    *p = 41;
    std::free(p);
    CHECK(*p == 41);                    // ASan aborts here (reached on Windows, where UBSan didn't abort)
}
```

`tests/CMakeLists.txt` — add the source to the existing runner (only for the proof; reverted later):

```cmake
add_executable(aero_tests
    test_main.cpp
    sanity_test.cpp
    red_proof_test.cpp
)
```

**Case-order rationale (§3.2 constraint 4):** doctest runs cases in registration order and a sanitizer abort kills the whole process. Ordering CHECK → UBSan → ASan means one Debug process records the CHECK failure, then trips the first sanitizer reachable per platform. Reversing it would hide UBSan on mac/Linux.

Expected per-lane outcomes (record on the PR):

| Lane | Debug step | Release step |
|---|---|---|
| macOS / Linux | red — CHECK failure recorded, then **UBSan abort** at case 2 (stacktrace via `UBSAN_OPTIONS`) | red — CHECK failure only (no sanitizers in Release; the UB cases pass silently — exactly the gap sanitizers cover) |
| Windows | red — CHECK failure recorded, case 2 passes silently (no UBSan — documented gap), then **ASan abort** at case 3 | red — CHECK failure only |
