1. `.github/workflows/ci.yml` exists and is valid (parses; `actionlint` clean).
2. On every push to `main` and every PR, a matrix runs on **`ubuntu-24.04` + `macos-15` + `windows-2022`**.

4. **vcpkg binary caching** is active (`actions/cache`): a warm run restores SDL3 instead of recompiling from source.
5. The **Windows** lane sets up the MSVC environment; the **Ubuntu** lane `apt install`s SDL3's system deps before configure.
6. The **baseline-invariant guard** job fails the workflow if `git rev-parse HEAD:vcpkg` ≠ `builtin-baseline`.
7. A **CI status badge** (default-branch state) is in `README.md`; `CLAUDE.md`'s "Current state" note is updated to record that CI landed.
8. **Criterion 7 from 0.1.1/0.1.2 is closed:** the Windows and Linux presets actually configure+build green in CI (proven on this task's own PR run).

## 3. Design — the artifact set (exact content)

Three files: one new workflow, two edits.

### 3.1 `.github/workflows/ci.yml` (new)

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

# One in-flight run per branch/PR; new pushes cancel the old run.
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

# Least privilege — this workflow only reads the repo.
permissions:
  contents: read

jobs:
  # Fast guard: the pinned vcpkg submodule and the manifest baseline must not drift.
  vcpkg-baseline:
    name: vcpkg baseline invariant
    runs-on: ubuntu-24.04
    steps:
      - name: Checkout (gitlink only — no submodule contents needed)
        uses: actions/checkout@v4
      - name: Submodule SHA must equal builtin-baseline
        shell: bash
        run: |
          submodule_sha="$(git rev-parse 'HEAD:vcpkg')"
          baseline="$(jq -r '.["builtin-baseline"]' vcpkg.json)"
          echo "submodule gitlink   : $submodule_sha"
          echo "vcpkg.json baseline : $baseline"
          if [ "$submodule_sha" != "$baseline" ]; then
            echo "::error file=vcpkg.json::vcpkg submodule SHA ($submodule_sha) != builtin-baseline ($baseline). Bump them together in one commit."
            exit 1
          fi

  build:
    name: ${{ matrix.name }}
    needs: vcpkg-baseline        # don't spend build minutes if the pin is broken
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false            # one OS failing must not cancel the others
      matrix:
        include:
          - name: Linux (GCC)
            os: ubuntu-24.04
            preset: linux
            vcpkg_cache: ~/.cache/vcpkg/archives
          - name: macOS (AppleClang)
            os: macos-15
            preset: macos
            vcpkg_cache: ~/.cache/vcpkg/archives
          - name: Windows (MSVC)
            os: windows-2022
            preset: windows
            vcpkg_cache: ~/AppData/Local/vcpkg/archives
    steps:
      - name: Checkout with vcpkg submodule
        uses: actions/checkout@v4
        with:
          submodules: true         # required — the toolchain lives in the submodule

      - name: Install SDL3 system dependencies (Linux)
        if: runner.os == 'Linux'
        run: |
          sudo apt-get update
          sudo apt-get install -y \
            libx11-dev libxft-dev libxext-dev libwayland-dev \
            libxkbcommon-dev libegl1-mesa-dev libibus-1.0-dev

      - name: Set up MSVC environment (Windows)
        if: runner.os == 'Windows'
        uses: ilammy/msvc-dev-cmd@v1   # consider SHA-pinning this third-party action (§6)
        with:
          arch: x64

      - name: Cache vcpkg binary archives
        uses: actions/cache@v4
        with:
          path: ${{ matrix.vcpkg_cache }}
          key: vcpkg-${{ matrix.os }}-${{ hashFiles('vcpkg.json') }}
          restore-keys: |
            vcpkg-${{ matrix.os }}-

      - name: Configure & build — Debug
        shell: bash                 # -eo pipefail: a failed configure aborts before build
        run: |
          cmake --preset ${{ matrix.preset }}-debug
          cmake --build --preset ${{ matrix.preset }}-debug

      - name: Configure & build — Release
        shell: bash
        run: |
          cmake --preset ${{ matrix.preset }}-release
          cmake --build --preset ${{ matrix.preset }}-release
```

## 4. Approaches considered & tradeoffs

### 4.1 vcpkg caching (D1)

| | A. `actions/cache` over the `files` dir *(chosen)* | B. vcpkg `x-gha` backend | C. NuGet feed (GitHub Packages) |
|---|---|---|---|
| Availability | ✅ First-party action, supported | ❌ **Removed upstream** (verified) | ✅ Supported |
| Setup | ✅ No tokens; one `path`/`key` | — | ⚠️ Feed + auth + `nuget.config` |
| Fit for 1 dependency | ✅ Whole-cache key is fine | — | ⚠️ Over-engineered now |
| Correctness on stale hit | ✅ vcpkg re-validates ABI, rebuilds if needed | — | ✅ same |

**A chosen** — B is not an option; C is heavyweight for a single dependency. MS docs themselves recommend pairing vcpkg's `files` cache with the CI's native cache.

### 4.2 Tool provisioning (D2)

| | A. Preinstalled + `msvc-dev-cmd` *(chosen)* | B. `lukka/get-cmake` on every OS |
|---|---|---|
| CMake/Ninja | ✅ Verified present (3.31 / 1.13.2) | ✅ Installs pinned versions |
| Moving parts | ✅ One unavoidable third-party action | ⚠️ Extra floating action + re-install |
| Value now (no targets) | ✅ Sufficient | ❌ Pins versions nothing compiles yet |

**A chosen.**

### 4.3 Matrix shape (D5)

| | A. 3 OS lanes, Debug+Release sequential *(chosen)* | B. 6 jobs (os × config) |
|---|---|---|
| SDL3 builds | ✅ Once per lane | ⚠️ Up to twice per OS on cold cache |
| Wall-clock | ⚠️ Slightly longer per lane | ✅ More parallel |
| Runner usage | ✅ Lower | ❌ Higher |

**A chosen** — SDL3 build cost dominates; sharing it per lane is the win.

### 4.4 Runner pinning (D3)

Pinned *(chosen)* vs `-latest` — pinning matches the project's pin-everything ethos and makes the verified image facts (§ external-facts) exactly what runs; the cost is a deliberate label bump when GitHub retires an image (announced months ahead).

- **`x-gha` removed:** MS Learn `consume/binary-caching-github-actions-cache` renders a "The GitHub Actions Cache backend for binary caching has been removed" caution → `microsoft/vcpkg-tool` PR #1662. This is why D1 = `files` + `actions/cache`.
- **Default `files` cache locations:** confirmed on MS Learn `users/binarycaching` (locations quoted in the external-facts header). Hence the per-OS `vcpkg_cache` matrix values.
- **Runner preinstalls (actions/runner-images READMEs):** `ubuntu-24.04` → CMake 3.31.6, Ninja 1.13.2. `macos-15` → CMake 3.31.5, Ninja 1.13.2, Xcode 16.4 (Apple Clang). `windows-2022` → CMake 3.31.6, Ninja 1.13.2, VS 2022 Enterprise 17.14 with MSVC v143. Hence D2 (no setup action needed except MSVC env).

## 6. Edge cases & operational notes

1. **Missing `submodules: true` ⇒ every configure fails** ("toolchain file not found"). The single most important line; called out in the 0.1.2 handoff.
2. **First run is slow / online** — vcpkg compiles SDL3 from source (minutes) until the cache warms. Expected, not a failure. Run #2+ restores from the binary cache.
3. **`msvc-dev-cmd` env crosses shells** — it writes to `GITHUB_ENV`/`GITHUB_PATH`, which GitHub applies to *all* later steps, so the `shell: bash` build steps inherit `cl.exe`/`INCLUDE`/`LIB`.
4. **Multi-line `run` fail-masking** — avoided by `shell: bash` (`-eo pipefail`): a failed `cmake --preset` aborts the step before the build line runs.
5. **`runner.os` conditionals** (`'Linux'`/`'Windows'`) rather than the pinned label — survive a future image bump (D3) without touching the `if:`.
6. **Cache key busts correctly** — `hashFiles('vcpkg.json')` changes on any dependency/feature/**baseline** edit; the baseline moves in lockstep with the submodule (the guard enforces it), so the key tracks the real inputs. `restore-keys` gives a safe partial hit.
7. **jq is preinstalled** on `ubuntu-24.04` — used by the guard job to read `builtin-baseline`.

| A pinned image is retired by GitHub | Low | Med | Announced months ahead via workflow annotations; bump the label deliberately (same shape as a vcpkg baseline bump) |
| `msvc-dev-cmd` env not seen by the bash step | Low | Med | Action writes to `GITHUB_ENV`/`GITHUB_PATH`; applied to all later steps regardless of shell |
| Third-party action supply chain (`msvc-dev-cmd`) | Low | Med | Optionally SHA-pin (with a version comment); official `actions/*` stay on major tags |

**Known deferral (tracking-only — NOT 0.1.3 scope).** `docs/04` states the compiler policy "**Clang on macOS/Linux**, MSVC on Windows, **GCC as Linux secondary**." 0.1.3 runs the OS matrix with each runner's *default* compiler, so the Linux lane uses **GCC** (docs/04's "secondary"), not Clang-primary, and adds **no** second Linux lane. Reason: with **zero compiled engine targets**, forcing Clang-primary would require a custom Clang Linux triplet (so vcpkg builds SDL3 with Clang too) for no present benefit, and the 0.1.3 subtasks say nothing about compilers. **Recommendation:** resolve the Linux compiler matrix at **0.2.1** (first `core` target) or as its own small follow-up — same species as the clang-format gap that became 0.1.6.
