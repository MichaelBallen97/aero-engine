## 1. Context & current state

- **0.1.4 is done and merged; `main` is green.** The repo is a build skeleton with exactly one compiled target — `aero_tests` (doctest) — built and tested (Debug ASan/UBSan + Release) on all 3 OSes in CI. `engine/` subsystem dirs are `.gitkeep`-only; `engine/CMakeLists.txt` and siblings are comment-only placeholders.
- **This task adds two firsts:** the engine's **first header** (`engine/core/include/aero/core/profiler.hpp`) and the **second `cmake/` module** (`cmake/profiling.cmake`, after `cmake/sanitizers.cmake`). It introduces the `aero::profiling` INTERFACE library and the project's first non-test dependency wired into a target.
- **Why this task exists:** `docs/04` "profile early, not after" — Tracy must be a habit before subsystems land, and the Phase 8 forward+ vs deferred render decision is explicitly gated on *real Tracy captures* (`docs/08-risks.md:31`, `docs/05-roadmap.md:168`). Tracy is **dev-builds-only** — an architecture invariant (`CLAUDE.md:39`, `docs/01:81`, `docs/03:91`), so it must be linked only under an explicit, default-OFF switch and never reach a shipped runtime.
- **It closes the "Tracy → 0.1.5" handoffs** deferred by every prior epic-0.1 task (`docs/plans/0.1.{1,2,4}` out-of-scope tables).

### Acceptance criteria

1. **Manifest:** `vcpkg.json` gains a plain `"tracy"` dependency (no features); `builtin-baseline` unchanged (0.13.1 at the pin, fact **a**); the CI `vcpkg-baseline` guard stays green.
2. **Option + module:** `AERO_ENABLE_PROFILING` (default OFF) and `cmake/profiling.cmake` exist; the module defines `aero_profiling` / `aero::profiling` (INTERFACE) unconditionally and, only when ON, `find_package(Tracy CONFIG REQUIRED)` + links `Tracy::TracyClient` + defines `AERO_PROFILING_ENABLED`.
3. **Preset mapping (D1):** the three `*-release` configure presets set `AERO_ENABLE_PROFILING=ON`; the three `*-debug` presets do not. Release compile commands for the `aero_tests` TU carry `AERO_PROFILING_ENABLED` (+ `TRACY_ENABLE` transitively); Debug compile commands carry **neither**.
4. **Wrapper header:** `engine/core/include/aero/core/profiler.hpp` exposes `AERO_PROFILE_*` macros that expand to Tracy when `AERO_PROFILING_ENABLED` is defined and to `((void)0)` otherwise; `<tracy/Tracy.hpp>` is included **only** when enabled; **no Tracy type appears in the API** (boundary rule).
5. **Live zone (D2):** `tests/profiler_test.cpp` calls the wrappers (a named zone + a frame mark); `aero_tests` links `aero::profiling`; the test passes on all six presets — no-op path in Debug, real Tracy path in Release — with **no profiler attached** (the Release binary exits promptly).
6. **Dev-builds-only invariant:** a default configure (`cmake .` with no preset / any `*-debug` preset) links **zero** Tracy and requires no Tracy package; Tracy is linked **only** in the `*-release` presets.
7. **CI:** all 3 OSes stay green — Debug (ASan/UBSan, Tracy off) + Release (Tracy on) configure/build/test. No YAML change is expected; **only** if the Linux Release link needs a system `-dev` lib is one added to the apt step (edge case E5).
8. **Docs (D8):** `CLAUDE.md` "Current state", a `README.md` Profiling note, and the `docs/04` profiling line are updated.
9. **Local verification** (§6) is green on the Mac before any push; **push + PR only after explicit user OK**.

### 3.2 `cmake/profiling.cmake` (new)

```cmake
# cmake/profiling.cmake — Tracy profiler wiring (docs/01 stack, docs/03 placement; task 0.1.5).
# Provides engine profiling via the aero::profiling INTERFACE library; macro wrappers live in
# engine/core/include/aero/core/profiler.hpp. Tracy is DEV-BUILDS-ONLY (docs/03 invariant): the
# client is linked and TRACY_ENABLE activated ONLY when AERO_ENABLE_PROFILING=ON (the *-release
# presets set it; OFF by default). When OFF, the wrappers compile to nothing and no Tracy is linked.
# Included from the root CMakeLists BEFORE add_subdirectory() so the target is globally visible.

add_library(aero_profiling INTERFACE)
add_library(aero::profiling ALIAS aero_profiling)

# Permanent home of the wrapper header (a core service like log/time; engine/core's compiled
# library lands at 0.2.1). Exposed to every consumer as <aero/core/profiler.hpp>, in all configs.
target_include_directories(aero_profiling INTERFACE ${CMAKE_SOURCE_DIR}/engine/core/include)

if(AERO_ENABLE_PROFILING)
    # Tracy 0.13.1 (pinned baseline). find_package only when enabled — a default build never needs it.
    # The vcpkg client is built with TRACY_ENABLE=ON (upstream default, PUBLIC), so linking the target
    # activates the Tracy macros in our code automatically.
    find_package(Tracy CONFIG REQUIRED)
    target_link_libraries(aero_profiling INTERFACE Tracy::TracyClient)
    # Drives the wrapper header; keeps the header backend-agnostic (only this module names Tracy).
    target_compile_definitions(aero_profiling INTERFACE AERO_PROFILING_ENABLED)
endif()
```

### 3.3 `engine/core/include/aero/core/profiler.hpp` (new — first engine header)

```cpp
#pragma once
// Aero Engine — profiling macro wrappers (task 0.1.5).
// Engine's own profiling API; the Tracy backend is an implementation detail that never crosses this
// boundary (project boundary rule). All macros compile to nothing unless the build enables profiling
// (AERO_ENABLE_PROFILING=ON → aero::profiling defines AERO_PROFILING_ENABLED). Tracy is dev-builds-only.
// Macros are statement-level: use at the top of a braced scope, not after a braceless if/for.

#if defined(AERO_PROFILING_ENABLED)
    #include <tracy/Tracy.hpp>
    #define AERO_PROFILE_ZONE                          ZoneScoped                 // scope, auto-named by function
    #define AERO_PROFILE_ZONE_NAMED(name)              ZoneScopedN(name)          // scope, literal name
    #define AERO_PROFILE_ZONE_NAMED_COLOR(name, color) ZoneScopedNC(name, color)  // name + 0xRRGGBB
    #define AERO_PROFILE_FRAME_MARK                    FrameMark                  // once per frame
    #define AERO_PROFILE_FRAME_MARK_NAMED(name)        FrameMarkNamed(name)
    #define AERO_PROFILE_PLOT(name, value)             TracyPlot(name, value)     // numeric graph
    #define AERO_PROFILE_MESSAGE(text)                 TracyMessageL(text)        // literal message
    #define AERO_PROFILE_SET_THREAD_NAME(name)         tracy::SetThreadName(name)
#else
    #define AERO_PROFILE_ZONE                          ((void)0)
    #define AERO_PROFILE_ZONE_NAMED(name)              ((void)0)
    #define AERO_PROFILE_ZONE_NAMED_COLOR(name, color) ((void)0)
    #define AERO_PROFILE_FRAME_MARK                    ((void)0)
    #define AERO_PROFILE_FRAME_MARK_NAMED(name)        ((void)0)
    #define AERO_PROFILE_PLOT(name, value)             ((void)0)
    #define AERO_PROFILE_MESSAGE(text)                 ((void)0)
    #define AERO_PROFILE_SET_THREAD_NAME(name)         ((void)0)
#endif
```
> The executor must confirm the macro spellings against the **installed** `<tracy/Tracy.hpp>` (0.13.1) after the first `macos-release` configure (`build/macos-release/vcpkg_installed/*/include/tracy/Tracy.hpp`); the set above is Tracy's stable public API (fact **d**). Adjust only if the installed header differs.

### 3.4 `tests/profiler_test.cpp` (new)

```cpp
// Exercises the profiling macro wrappers (task 0.1.5): they must compile, link, and run in BOTH
// configs — no-op when profiling is OFF (Debug presets), real Tracy zones when ON (Release presets).
// No Tracy Profiler needs to be attached; with none connected the client idles and the process exits.
#include <doctest/doctest.h>
#include <aero/core/profiler.hpp>

namespace {
int instrumented_work() {
    AERO_PROFILE_ZONE_NAMED("aero::tests::instrumented_work");
    int sum = 0;
    for (int i = 0; i < 1000; ++i) sum += i;
    return sum;
}
} // namespace

TEST_CASE("profiling: macro wrappers compile, link, and run in both configs") {
    CHECK(instrumented_work() == 499500);
    AERO_PROFILE_FRAME_MARK;   // must be a valid statement in both configs
}
```

## 5. Edge cases & risks

- **E1** Baseline provides exactly tracy 0.13.1 → no override; baseline unchanged → guard green *(verified, fact a/e)*.
- **E2** vcpkg client enabled by default (`TRACY_ENABLE` ON/PUBLIC) → linking `Tracy::TracyClient` activates the macros *(verified, fact c)*.
- **E3** Keep `gui-tools`/`cli-tools` OFF — plain `"tracy"` only; never add features (glfw3/freetype/curl/capstone/imgui blowup).
- **E4** The Release `aero_tests` must exit promptly in CI with no profiler connected (expected — the client idles, doesn't block). Verify first run; **never set `TRACY_NO_EXIT` in CI** (it would hang `ctest`).
- **E5** The Linux Tracy-client link may need a system `-dev` lib → conditional apt add (a `ci:` commit). Verify first CI run.
- **E6** Windows x64 supported (`supports: !(windows & (arm|uwp))`); the existing `rm -f /usr/bin/link` MSYS-shadow fix already covers the new linked lib.
- **E7** Boundary rule upheld: no Tracy type in the wrapper API; `<tracy/Tracy.hpp>` pulled only when enabled; every default/Debug build links zero Tracy → dev-builds-only holds by default-OFF + link-time gating.
- **E8** The interface lib is defined in both states (populated only when ON) so consumers link `aero::profiling` unconditionally.
- **E9** vcpkg *installs* tracy on every preset (manifest mode) but it is *linked* only in Release — harmless (small, cached); the invariant is about linkage, not installation. Manifest-feature gating is a possible future refinement.
- **E10** Macros are statement-level (like Tracy's) — documented "use at the top of a braced scope".
